# 接口与交互规范

<!--
  分层定位（重要）：
  本文件是**接口层的长期约束**——记录"我们所有对外接口必须遵守的长期规范"
  （命名约定、错误码体系、版本策略、交互模式等），属于架构层。
  它**不是**某次功能的技术设计；后者应放在 `docs/design-docs/{功能}.md`。

  修改本文件应被视为架构层变更，需要走独立的架构 RFC；
  不能在某个功能的执行计划里顺手改。

  本文件语言与框架无关，适用于前端/后端（Java/Python/Go/Node）/
  Android/iOS/CLI/SDK 等任意项目类型。

  不同类型的项目，"接口"的含义不同：
  - Web 前端项目：UI 设计系统（组件库、主题、交互模式）
  - 后端 API 项目：API 设计规范（URL 风格、请求/响应格式、错误码）
  - 移动端（Android/iOS）：页面导航、组件规范、平台交互模式
  - CLI 工具：命令行交互规范（命令结构、参数风格、输出格式）
  - SDK/库：公共 API 设计（命名规范、参数约定、版本策略）
  - 微服务：服务间通信协议（消息格式、事件 Schema、RPC 约定）

  使用指引：
  - 按项目类型**保留并填写相关章节**，**删除不适用的章节**（鼓励删除）
  - 初次创建时可以只留 1-2 个章节，其余留白或删除
  - 智能体在创建或修改任何对外接口前应先阅读此文件
-->
